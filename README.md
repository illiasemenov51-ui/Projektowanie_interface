# Projektowanie_interface

Opis projektu – Projektowanie interfejsu użytkownika (Figma)
1. Cel projektu

Celem projektu było zaprojektowanie intuicyjnego, przejrzystego i funkcjonalnego interfejsu użytkownika (UI) z wykorzystaniem narzędzia Figma. Projekt interfejsu ma wspierać działanie systemu wbudowanego / aplikacji, ułatwiać obsługę użytkownikowi oraz zapewniać czytelną prezentację informacji.

2. Zakres projektu

W ramach projektu wykonano:

Analizę potrzeb użytkownika (User Needs).

Opracowanie architektury informacji (Information Architecture).

Stworzenie kilku wariantów layoutów.

Zaprojektowanie ostatecznego interfejsu w Figma.

Przygotowanie interaktywnego prototypu.

Dostosowanie UI do responsywności (jeśli dotyczy).

3. Narzędzia i technologie

Do realizacji projektu wykorzystano:

Figma – projektowanie UI/UX, prototypowanie,

Systemy siatek (grid, auto-layout),

Komponenty, style i warianty,

Ikony z bibliotek (Material Icons / Feather Icons),

Kolorystykę opartą o zasady kontrastu WCAG.

4. Opis stworzonego interfejsu

Interfejs został zaprojektowany zgodnie z zasadami ergonomii:

Minimalistyczny i czytelny layout,

Wyraźna hierarchia wizualna,

Zastosowanie konsekwentnych kolorów i stylów,

Logiczne rozmieszczenie elementów (przyciski, panele, wskaźniki),

Uproszczony dostęp do najważniejszych funkcji,

Elementy interakcji zaprojektowane z użyciem prototypowania.

Przykład (можешь заменить своим):

Strona główna z panelem informacji,

Ekran ustawień systemu,

Dashboard czujników (np. temperatura, wilgotność),

Panel alertów,

Ekran statystyk.

5. Proces projektowy (UI/UX)
5.1. Analiza problemu

Zidentyfikowano potrzeby użytkownika, określono scenariusze użytkowania (User Stories), np.:

„Użytkownik chce szybko sprawdzić aktualne dane sensoryczne.”

„Użytkownik musi łatwo zmienić ustawienia systemu.”

5.2. Wireframes

Stworzono szkice (low-fidelity wireframes) prezentujące układ elementów i podstawową logikę ekranu.

5.3. High-fidelity design

W Figma wykonano finalny projekt z kolorami, ikonami i komponentami.

5.4. Prototyp

Interfejs połączono w interaktywny prototyp, umożliwiający symulację działania aplikacji.

6. Wnioski

Projekt interfejsu pozwolił na zdobycie praktycznych umiejętności z zakresu UI/UX oraz obsługi programu Figma. Opracowany interfejs jest estetyczny, spójny, intuicyjny i spełnia założenia funkcjonalne systemu.
